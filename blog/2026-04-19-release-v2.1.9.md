Stratora v2.1.9 changes how the Stratora tray indicator is registered for auto-start so it launches for every interactive Windows user after agent or collector installation. In-place upgrade from v2.1.8 or earlier.

<!-- truncate -->

## Fixed

- **Tray indicator starts automatically for all users** — The Stratora tray indicator now launches for every interactive Windows user at logon after agent or collector installation, using a per-machine startup registration. Previous releases had a single-user registration path that left the indicator invisible on multi-admin servers.

- **Orphan registry values cleaned up on upgrade** — Upgrading from a prior release automatically removes the leftover registry values created by the previous single-user registration path. No manual registry cleanup is required.

- **Single-instance guard on the tray indicator** — If a host carries a legacy per-user auto-start entry alongside the new per-machine entry during the upgrade window, the tray indicator deduplicates within a logon session so users see exactly one icon.

## Operator Notes

- In-place MSI upgrade from v2.1.8 or earlier is supported. Users will see the tray indicator on their next logon.
- If you rely on scripted invocations of `stratora-tray.exe -install-startup` or `-uninstall-startup`, those flags have been removed. Remove the scripted calls — auto-start registration is handled by the MSI installer in every case.
