Stratora v2.1.8 closes the last remaining gaps in auto-generated topology map behavior. The map is now a live mirror of the site's node set: adds, removes, and cross-site moves all propagate immediately, and three node-creation paths that previously bypassed the auto-generation chain now fire it correctly. First 2.1.8 boot cleans up drift accumulated under the previous additive-only behavior.

<!-- truncate -->

## The always-sync gap

Auto-generated topology maps shipped with create-once, additions-only behavior: a site's first node add triggered map creation, and subsequent adds got placed on the map, but node **removals** had no code path. Deleting a node, rejecting it, deactivating it, or moving it to another site all left behind an orphan element pinned to a nonexistent or out-of-site node. Over time, the map diverged from reality.

v2.1.8 replaces the additive-only sync with a full **reconcile** pass: for each invocation, it reads the site's current approved+active node set, prunes elements whose node no longer qualifies, and adds elements for nodes not yet on the map. Idempotent and safe to call at any time.

## The three bypass paths

A separate class of bugs surfaced during 2.1.7 validation: three node-creation surfaces created nodes directly without firing the auto-generation chain. These were invisible on dev environments where the bypassed paths weren't exercised, but broke greenfield installs where discovery imports or agent auto-enrollment was the first path to add nodes.

The three fixed in 2.1.8:

- **Discovery single-device import** — single-device imports from the discovery scanner now fire the topology reconcile and dashboard regeneration on each import.
- **Discovery bulk import** — bulk imports from discovery collect unique site IDs across the batch and fire one reconcile per affected site rather than per node. A 500-node bulk import into one site runs one reconcile pass, not 500.
- **Agent self-enrollment** — bootstrap-enrolled agents go straight to approved + active and now land on the site's auto-map immediately.

## The asymmetric hook on cross-site moves

Editing a node's site assignment and the bulk Move action both already fired hooks for the old and new sites, but under the additive-only behavior the old-site fire was a no-op (nothing removed the moved-out node's element from the old site's map). With reconcile live, these fires now correctly prune the element from the old site's map and add it to the new site's map.

## Topology maps now cascade-delete with their site

Topology maps now cascade-delete with their site. Auto-maps are site-intrinsic — deleting the site should take the map with it, not leave a ghost row. The upgrade also runs a one-shot cleanup of already-orphaned auto-maps from earlier releases. User-created maps are explicitly preserved even if their site is gone, so user-authored layouts aren't lost.

## First-boot reconcile cleanup

On first 2.1.8 boot, every site with nodes is reconciled once — not just sites missing a map. Accumulated orphan elements from every previous node deletion, rejection, or cross-site move are cleaned up in one pass. Customers running 2.1.x will see reduced element counts on their auto-maps after the restart, and those counts will match their live approved + active node sets.

## Operator notes

- First 2.1.8 startup runs the schema change and the reconcile-all-sites backfill automatically. No manual database intervention.
- Reconcile touches only auto-generated maps. User-edited maps (those a user has created or modified in the Map Editor) are not affected.
- Agent and Collector binaries are unchanged from 2.1.6. The Server MSI embeds the same Agent and Collector MSIs as the 2.1.7 release.
