Stratora Agent v2.1.2 is a Windows-only point release that adds per-scope DHCP utilization monitoring for Windows Server hosts running the DHCP Server role. The Stratora Server and Collector are unchanged at 2.1.0.

<!-- truncate -->

### DHCP Scope Utilization

Windows Performance Counters expose DHCP _message-rate_ metrics (Discovers/Offers/Acks per second, queue length), but not scope _address-pool_ utilization. Running out of addresses in a DHCP scope is one of the most disruptive failures a Windows DHCP server can suffer — new clients silently fail to obtain a lease — yet it was invisible to the agent until now.

The 2.1.2 agent ships with a new bundled PowerShell collector, `dhcp_scopes.ps1`, invoked by Telegraf on a 5-minute interval. The script calls `Get-DhcpServerv4Scope` and `Get-DhcpServerv4ScopeStatistics` locally (no `-ComputerName`) and emits two series per scope:

- `dhcp_scope_info` — all fields carried as string tags so a table panel can map columns directly to label keys (one row per scope)
- `dhcp_scope` — numeric fields (`addresses_in_use`, `addresses_free`, `addresses_reserved`, `addresses_pending`, `addresses_total`, `percent_in_use`) as first-class metrics for alerting

### What you see in the UI

Windows Server node detail pages now have a **DHCP Scope Utilization** section with a full-width table showing every scope on the server:

| Scope Name | Scope ID | In Use | Free | Reserved | Pending | Total | % Used |
|---|---|---|---|---|---|---|---|

The **% Used** column is color-coded using the Stratora health palette — green below 75%, yellow from 75% to 89%, red at 90% or higher — so exhausted scopes jump out visually before they hit the alert threshold.

### New built-in alerts

Two new built-in alert definitions ship with the agent release (server-side migration 148):

- **DHCP Scope High Utilization** — warning at 80%, critical at 90%
- **DHCP Scope Exhaustion** — critical at 90%

Both fire against the `dhcp_scope_percent_in_use` metric. Because the alert evaluator currently collapses multi-series metrics to the worst value per node, a DHCP server with multiple scopes will receive a single alert keyed on the worst-utilized scope rather than one alert per scope. Per-scope independent alerting is on the post-GA backlog.

### Zero-impact on non-DHCP hosts

The same agent MSI is safe to deploy to every Windows server regardless of installed roles. The collector script exits `0` silently when the `DhcpServer` PowerShell module is not installed, so File Servers, Web Servers, and Domain Controllers without the DHCP role see nothing new in their Telegraf logs and collect no DHCP metrics.

### Upgrade

Redeploy `StratoraAgent-2.1.2.msi` over the existing installation on any Windows server running DHCP:

```powershell
msiexec /i StratoraAgent-2.1.2.msi /qn
```

The MSI uses an in-place MajorUpgrade — configuration and enrolled component credentials are preserved, so the agent does not re-enroll and the node record is retained. Wait one 5-minute collection interval after upgrade for scope metrics to appear on the node detail page.
