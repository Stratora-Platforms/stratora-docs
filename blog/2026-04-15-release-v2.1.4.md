Stratora v2.1.4 is a hotfix release that resolves a first-run experience issue where the setup page would hang indefinitely instead of redirecting to the login screen.

<!-- truncate -->

### Setup Page Fix

On fresh installations, the `/setup` splash page displays "Getting things ready..." while backend services initialize, then automatically redirects to `/login` once ready. In previous versions, the JavaScript polling loop responsible for this redirect was embedded as an inline script in the HTML page. Because NGINX enforces a strict `script-src 'self'` Content Security Policy, the browser silently blocked the inline script — leaving the page stuck on "Getting things ready..." forever.

This release extracts the polling script into a separate embedded file (`setup.js`) served from a dedicated route. The script now loads as an external resource, which satisfies the CSP policy. The auto-redirect to `/login` after service initialization works correctly on all fresh installs.

### Upgrade Notes

Existing installations can upgrade by running the new `Stratora-Server-2.1.4.msi` installer over the existing installation. This is a recommended upgrade for all users, particularly those deploying new Stratora instances.
