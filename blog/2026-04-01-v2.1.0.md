Stratora v2.1.0 is now generally available. This release adds disk capacity reporting, site photos, a dedicated site detail page, and several quality-of-life improvements across the platform.

<!-- truncate -->

### Disk Capacity Reporting

A new **Disk Capacity** report type is available in the Reports engine. It generates PDF reports showing disk usage trends across your monitored nodes, with projected full dates calculated using linear regression. Reports can be run on demand or delivered on a schedule, just like existing report types.

### Site Photos

You can now upload and manage photos for each site — equipment shots, rack layouts, floor plans, or anything else worth documenting. Photos are stored securely, served with authentication, and viewable in a lightbox gallery on the site detail page.

### Site Detail Page

Each site now has a dedicated page with eight tabs: **Overview**, **Dashboard**, **Nodes**, **Topology**, **Racks**, **Alerts**, **Reports**, and **Photos**. The Overview tab shows node health, device breakdown, and IPAM networks at a glance. An attention strip at the top surfaces degraded, critical, and offline nodes so you can assess site status immediately.

### Node Quick Filters

The Nodes tab on the site detail page includes filter toggles — **All**, **Warning**, **Critical**, **Offline**, and **Healthy** — each with a live count. This lets you focus on problem nodes without leaving the site context.

### About Page

**Settings → About** now displays the running platform version, build information, and open source component attribution.

### Bug Fixes & Polish

- Platform version is now correctly reported across all API endpoints
- Homepage layout streamlined — cleaner health ring, reordered right rail, reactive accent bar and ambient glow reflecting node health state
- Report template names normalized for consistency
