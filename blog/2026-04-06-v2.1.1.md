Stratora v2.1.1 is a maintenance release. It adds three built-in report types, Linux CPU and memory alert definitions, and fixes several issues in the discovery, alerting, and NGINX reload paths.

<!-- truncate -->

### New Built-In Reports

Three new report types are available out of the box:

- **Alert Intelligence** (7-day and 30-day) — severity breakdown, top noisy nodes, time-to-acknowledge and time-to-resolve metrics.
- **Availability / SLA** (monthly and quarterly) — per-site uptime percentage, downtime, incident count, MTTR, and MTBF.
- **Top Offenders** (24-hour and 7-day) — top nodes by CPU, memory, latency, and interface errors.

### Linux CPU and Memory Alerts

Built-in alert definitions for Linux nodes have been added: **High CPU Usage** (warn 80%, crit 95%) and **High Memory Usage** (warn 80%, crit 95%), each with a 5-minute sustain window. The alert evaluator now resolves `linux_cpu_usage_percent` and `linux_memory_usage_percent` queries scoped to `linux-server` nodes only.

The Windows and Linux memory alert warning thresholds have both been lowered from 85% to 80% in the built-in definitions.

### Fixes

- **NGINX reload no longer causes a 5-second HTTPS outage.** FQDN save, certificate upload, ACME issuance, and manual reload operations now restart the service asynchronously after the HTTP response is sent.
- **Setup wizard step 3** now shows an error toast when an FQDN save fails on a network error, instead of silently stalling.
- **Top Resources widget** no longer returns 500 — removed an invalid `deleted_at` column reference; the canonical `is_active = true` filter is sufficient.
- **Stuck discovery jobs** left in `running` state after a server crash or unclean shutdown are now automatically recovered at startup and marked failed with a descriptive error.
- **Discovery scan context** now has a 1-hour maximum runtime deadline. Scans that hang indefinitely are cancelled and marked failed.
- **Collector Failover alert** no longer fires false positives against non-collector nodes when they go offline. The definition is now correctly routed to the collector-scoped evaluator.
- **Alert toast ACK/Escalate buttons** now show "Alert already resolved" instead of a generic error when the alert resolves before the user can act.
